3. 로드 밸런싱 시스템의 요구사항, 부하 분산의 목적, 서비스 인스턴스의 특성 등을 고려하여 결정.
   1. 라운드 로빈 (Round Robin): 라운드 로빈 알고리즘은 요청을 순서대로 서비스 인스턴스에 분배(각 서비스 인스턴스가 동일한 처리 능력을 가정할 때 효과적)
   2. 가중치 기반 (Weighted): 가중치 기반 알고리즘은 서비스 인스턴스에 가중치를 할당하여 요청을 분배(각 서비스 인스턴스는 처리 능력이 다를 수 있으므로, 가중치를 설정하여 더 많은 요청을 처리할 수
      있는 서비스 인스턴스에 더 많은 트래픽을 전달 가능)
   3. 최소 연결 (Least Connection): 최소 연결 알고리즘은 현재 연결 상태를 고려하여 요청을 가장 적은 연결을 가진 서비스 인스턴스에 분배(각 서비스 인스턴스의 부하 상태를 고려하여 트래픽을
      분산하므로, 부하가 균일하게 분배)
   4. 가장 짧은 지연시간 (Least Response Time)
   5. IP 해싱 (IP Hashing)
   6. 랜덤 (Random)
4. 트래픽 관리
   - 요청 제한 (Rate Limiting): 단위 시간당 허용되는 최대 요청 수를 제한하여 트래픽을 제어 가능(ex. 초당 100개의 요청으로 제한할 경우 초과된 요청은 거부되거나 대기열에 추가)
   - 회로 차단기 (Circuit Breaker): 백엔드 서비스의 장애 상황을 감지하고, 일시적으로 해당 서비스와의 통신을 차단함으로써 전체 시스템에 영향을 줄이는 기능
   - 품질 기반 라우팅 (Quality of Service-based Routing)
   - 요청 변환 및 포매팅 (Request Transformation and Formatting): 클라이언트의 요청을 백엔드 서비스가 이해할 수 있는 형식으로 변환하거나, 요청 데이터의 포맷을 일관성 있게
     조정
   - 오류 처리와 재시도 (Error Handling and Retries): 백엔드 서비스에서 발생하는 오류를 처리하고, 재시도 (Fail Back)

6. API 모니터링과 로깅
   1. 트래픽 및 요청량 (Traffic and Request Volume)
      초당 요청 수, 요청 유형 및 트래픽 패턴 등을 포함. 트래픽의 증가 또는 감소 파악 및 서버의 부하 예측, 대응 가능.
   2. 응답 시간 (Response Time)
      응답 시간 모니터링은 성능 이슈를 감지하고, 지연이 발생하는 요청을 식별하여 최적화 가능.
   3. 에러 및 예외 (Errors and Exceptions)
      오류 응답의 유형, 발생 빈도, 원인 등. 오류 유형을 식별하고, 서비스의 안정성을 향상시키기 위해 필요한 조치 가능.
   4. 로그 (Logging)
      API 요청의 세부 정보, 요청 헤더, 응답 상태 코드, 오류 메시지 등을 포함. 로그 기록을 통해 디버깅, 오류 추적, 보안 감사 등 다양한 목적으로 활용 가능.
   5. 인증 및 권한 부여 (Authentication and Authorization)
      인증 실패, 접근 거부, 권한 부여 이벤트 등을 포함 인증 및 권한 부여 모니터링은 보안 상태를 파악하고, 잘못된 인증 시도나 권한 위반을 탐지.
   6. 서비스 가용성 (Service Availability)
      API Gateway 자체의 가용성, 연결된 백엔드 서비스의 가용성, 네트워크 연결 등을 포함 서비스 가용성. 모니터링은 장애 발생 여부를 식별하고, 서비스의 신뢰성과 안정성을 보장하기 위해 필요.
   7. 성능 지표 (Performance Metrics)
      응답 시간, 처리량, 대기 시간 등의 성능 지표를 포함. 성능 지표 모니터링은 서비스의 성능 향상을 위해 병목 현상을 파악하고 최적화 가능.
   8. 보안 이벤트 (Security Events)
      악성 요청, DDoS 공격 시도, 인증 실패 등 보안 위협에 관련된 이벤트.
   9. 요청 및 응답 헤더 (Request and Response Headers)
      요청 및 응답 헤더의 값, 크기, 캐싱 여부 등을 포함. 요청 및 응답 헤더 모니터링은 헤더 관련 이슈를 파악하고, 헤더 조작으로 인한 보안 문제를 탐지 가능.
   10. 서비스 경로 및 엔드포인트 (Service Paths and Endpoints)
       각 경로와 엔드포인트의 요청 수, 응답 시간, 성공률 등을 포함. 서비스 경로 및 엔드포인트 모니터링은 특정 경로 또는 엔드포인트에서의 이슈를 파악하고, 서비스 전반의 성능을 개선할 수 있도록 도와줌.
   11. 요청 및 응답 페이로드 (Request and Response Payload)
       요청과 응답의 데이터 크기, 형식, 구조 등을 포함. 페이로드 모니터링은 데이터 무결성, 데이터 유효성 검사, 너무 큰 페이로드로 인한 성능 저하 등을 식별.
   12. 비용 및 요금 청구 (Cost and Billing)
       각 API 호출에 따른 비용, 서비스 사용량, 요금 청구 내역 등을 포함. 비용 및 요금 청구 모니터링은 비용 최적화 및 예산 관리를 위해 필요한 정보를 제공.
   13. 실시간 모니터링 대시보드 (Real-time Monitoring Dashboard)
       시각화하여 제공하는 인터페이스 대시보드는 시스템 상태, 성능, 트래픽 패턴, 오류 등의 정보를 실시간으로 표시하여 관리자가 서비스 상태를 쉽게 파악하고 필요한 조치를 취할 수 있도록 도와줌.
   14. 알림 및 경고 시스템 (Alerting and Notification System)
       이메일, SMS, 푸시 알림 등의 방식으로 관리자나 운영팀에게 경고를 전달하는 기능. 알림 및 경고 시스템은 신속한 대응과 서비스 회복을 돕는 중요한 도구.
   15. 히스토리 및 추적 (History and Tracing)
       각 요청의 시작 시간, 종료 시간, 호출자 정보, 백엔드 서비스 연동 로그 등을 포함 히스토리와 추적 기능은 문제 해결, 디버깅, 오류 추적, 감사 및 규정 준수를 위해 필요.

8. 캐싱
   - Reverse Proxy 캐싱: API Gateway를 구성하는 프록시 서버가 응답을 캐시하여 다음에 동일한 요청이 들어올 경우 캐시된 응답을 반환 (ex. Nginx, Apache HTTP
     Server와 같은 웹 서버를 API Gateway로 사용하고, 해당 서버의 캐싱 기능을 활성화하여 캐싱을 구현)
   - 인메모리 캐싱: Redis, Memcached와 같은 인메모리 데이터베이스를 사용하여 캐싱 구현
   - 클라우드 기반 캐싱: Amazon CloudFront, Azure CDN, Google Cloud CDN과 같은 클라우드 기반의 콘텐츠 전송 네트워크 서비스를 사용하여 캐싱을 구현       서비스는 전 세계에 분산된 캐시 서버를 통해 콘텐츠를 캐시하고, 사용자에게 가장 가까운 서버에서 응답을 제공함으로써 빠른 응답 속도와 효율적인 트래픽 관리를 지원합니다.
